# Работа с элементом Grid(таблицы)

Элемент grid хорошо документирован с помощью demo примеров в [Kendo UI](https://demos.telerik.com/kendo-ui/grid/index). В faceplate мы можем использовать его следующим образом:

> 1. Использовать в Мнемосхемах
> 2. Использовать в Отчетах

В качестве источника данных мы можем использовать динамические и статические данные, которые могут быть добавлены с помощью Table Datasource, TimeSeries Datasource и JSON Datasource.

## Настройка таблицы с динамическим источником данных

В качестве примера возьмем список автомобилей с соответствующими динамическими и статическими параметрами, соответственно для каждого автомобиля. Необходимо получить таблицу с заголовками : **Name , Brand, Color, Max_Speed, Torque, Speed**

### Необходимые шаги

1. Cоздание папки
2. Создание нового шаблона
3. Создание тегов
4. Создание отчётов
5. Добавление элементов Table Datasource и Grid
6. Конфигурация элементов Table Datasource и Grid

**Создание нового шаблона**

> Создайте теги для каждого автомобиля, а для шаблона тега (шаблона) необходимо создать новый. Давайте создадим шаблон
> тега, для этого нужно выбрать шаблоны из редактор (есть два варианта: *теги, шаблоны*)

<kbd><img src="/img/0_2.png" width="650" height="350"></kbd>

> При добавлении поля, необходимо написать имя параметра, такого как «Brand», тип «string» и тип хранилища «ramdisc».
> Выбор между ramdisc или ram зависит от того, как значение параметра должно быть сохранено на жестком диске.
> Например, имя параметра не изменяется по времени часто и его необходимо сохранить на диск,
> в то время как параметр «Speed» меняется каждый раз.
> Так что наш шаблон "car" будет выглядеть так:
